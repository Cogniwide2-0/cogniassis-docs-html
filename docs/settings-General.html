General

When you click on "Configurations" and navigate to the "General" page, you'll find essential project settings neatly organized for easy management. This section allows you to set up foundational elements crucial for your AI bot's operation.Here, you can define the fallback response strategy, ensuring a seamless user experience even when specific queries aren't recognized.Here you can view the full list of supported General.

  1. General
  2. Machine Learning
  3. Users
  4. Configuration Data

By default, the REST channel is enabled to facilitate one-to-one communication with the bot. This setup provides a REST endpoint where messages can be posted, and the bot responds accordingly.

General "General" settings act as your AI project's control center. Here you define its core identity (title, description), language (default and optional others), and overall functionality, laying the foundation for how your bot interacts with users. It simplifies project management and offers a clear starting point for new users.

Machine Learning

Click the Machine Learning option from the left side menu of the screen. It displays the following screen.

Drag the pointer to the right side to increase the threshold limit and to the left side to decrease the threshold limit.

Note - If the threshold is very high, then the AI bot will respond only if the confidence level is high.

Users "User" in "General" settings can refer to two main types:

Your AI Project's Users: You might define the target audience (e.g., customer types) and access levels (e.g., read-only, admin) for users interacting with your bot.

Platform Users: More likely, it manages users within the development platform itself, like bot developers, project managers, and system admins. Manages user information to facilitate personalized interactions based on user profiles and historical data.
